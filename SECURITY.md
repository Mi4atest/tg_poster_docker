# Улучшения безопасности и отказоустойчивости

В этом документе описаны улучшения безопасности и отказоустойчивости, внесенные в проект TG Poster.

## Основные изменения

1. **Ограничение доступа к портам**
   - Закрыт прямой доступ к базе данных PostgreSQL (порт 5432)
   - Закрыт прямой доступ к API (порт 8002)
   - Настроен Nginx как единственная точка входа

2. **Настройка SSL/TLS**
   - Добавлена поддержка HTTPS с использованием SSL/TLS
   - Настроены современные протоколы шифрования
   - Добавлен HSTS для защиты от атак понижения уровня протокола

3. **Настройка Web Application Firewall (WAF)**
   - Добавлен ModSecurity для защиты от атак
   - Настроены правила для защиты от распространенных атак (XSS, CSRF, инъекции)

4. **Ограничение доступа по IP**
   - Настроен доступ только с IP-адресов Telegram API для webhook
   - Блокировка всех остальных запросов

5. **Настройка Webhook для Telegram**
   - Добавлена поддержка Webhook вместо Long Polling
   - Добавлена проверка подписи сообщений от Telegram

6. **Резервное копирование**
   - Добавлены скрипты для автоматического резервного копирования
   - Настроено шифрование резервных копий

## Установка и настройка

### 1. Генерация SSL-сертификатов

```bash
./generate_ssl.sh
```

Этот скрипт создаст самоподписанные SSL-сертификаты для локального использования. Для production рекомендуется использовать сертификаты от Let's Encrypt.

### 2. Настройка безопасности

```bash
./setup_security.sh
```

Этот скрипт настроит основные параметры безопасности, включая:
- Генерацию SSL-сертификатов (если они еще не созданы)
- Настройку ModSecurity
- Обновление переменных окружения с безопасными значениями

### 3. Настройка брандмауэра (требуются права root)

```bash
sudo ./setup_firewall.sh
```

Этот скрипт настроит брандмауэр UFW для:
- Разрешения доступа только к портам 80 и 443
- Разрешения доступа только с IP-адресов Telegram API для webhook
- Блокировки всех остальных портов

### 4. Настройка переменных окружения

Отредактируйте файл `.env` и установите следующие переменные:

```
# SSL/TLS
SSL_ENABLED=true
SSL_CERT_PATH=/etc/nginx/ssl/fullchain.pem
SSL_KEY_PATH=/etc/nginx/ssl/privkey.pem

# Webhook (для Telegram)
USE_WEBHOOK=true
WEBHOOK_URL=https://your-domain.com/api/telegram/webhook
WEBHOOK_SECRET=your_webhook_secret
```

### 5. Запуск контейнеров

```bash
./stop.sh && ./start.sh
```

## Резервное копирование и восстановление

### Создание резервной копии

```bash
./backup.sh
```

Этот скрипт создаст зашифрованные резервные копии базы данных и медиа-файлов в директории `backups`.

### Восстановление из резервной копии

```bash
./restore.sh ./backups/backup_20230101_120000.sql.enc ./backups/media_20230101_120000.tar.gz.enc your_password
```

## Проверка безопасности

После настройки рекомендуется проверить безопасность с помощью следующих инструментов:

1. **SSL Labs** - https://www.ssllabs.com/ssltest/
2. **Mozilla Observatory** - https://observatory.mozilla.org/
3. **Security Headers** - https://securityheaders.com/
